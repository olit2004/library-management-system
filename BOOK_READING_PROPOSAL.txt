
This proposal outlines the integration of the Google Books API and Embedded Viewer into the Library Management System (LMS). 

1. Objective
Leverage Google’s massive digital index to provide users with instant book previews and detailed metadata directly within the LMS interface, while keeping the option to link to full digital copies where available.
2. Technical OverviewA. API & Content Access

* Google Books API (REST): Used to fetch book details (synopsis, ratings, page count) via ISBN or Title.
* Google Embedded Viewer API (JS): A client-side library used to render a "Preview" window. This allows users to browse a limited number of pages (or full text if public domain) without leaving the app.
* Fallback Storage: The Server/public/books directory will now serve as a fallback only for rare or internal titles not found on Google.

B. Database Schema Changes
The Book model in schema.prisma will be updated to prioritize Google’s unique identifiers:

* google_volume_id: String (Unique ID from Google to fetch the correct preview).
* preview_link: String (Direct URL to the Google Books preview page).
* is_digital: Boolean (True if a preview or full version is available via the API).

C. Frontend Reader Component

* Google Viewer Component: A specialized React component that loads the Google jsapi. It will target a <div> in the /member/read/:id route.
* Dynamic Loading: The viewer will initialize using the google_volume_id.
* UI Integration: A "Preview Book" button will appear on the Discover page, launching an immersive overlay or dedicated reading page.

3. Proposed Implementation PhasesPhase 1: API Integration & Data Enrichment

   1. Obtain Google Cloud API credentials.
   2. Update the Book model and sync existing ISBNs with the Google Books API to populate google_volume_id and preview_link.
   3. Create a backend service that fetches real-time "Previewability" status from Google.

Phase 2: The "Embedded Reader" UI

   1. Build a BookViewer React component using a WebView or <iframe> strategy to handle the Google JS library.
   2. Implement "Partial Preview" logic: If a book has a preview available, show the "Read Preview" button; otherwise, show "Physical Copy Only."

Phase 3: Enhanced Discovery

   1. Use Google’s categories and averageRating to improve the Discover page.
   2. Implement "Search by Google": Allow admins to add new books to the LMS by simply entering an ISBN and auto-filling all details from the API.

4. Considerations & Constraints

* Preview Limits: Most modern books only allow a "Preview" (approx. 20-40% of the book). For full reading, the LMS must still point to a physical copy or a local PDF if the copyright allows.
* Client-Side Heavy: The viewer runs in the user's browser, meaning no server-side bandwidth is used for reading.
